## Validación obligatoria

Cada comando tiene que pasar antes de avanzar al build de ISO.
Sin `sudo` automático; los comandos que requieran instalación
local (`make check-installed`) se corren manualmente cuando se
decida instalar la versión.

```bash
make check
make check-installed
watson doctor
watson readiness
watson negative-tests
watson contracts check
watson tools list
watson report --mode desarrollo --out /tmp/openclaw-report.md
scripts/release-checklist.sh v0.4.0-alpha
```

Esperado:

- `make check`: `fails=0`.
- `make check-installed`: drift cero contra el repo (requiere
  `sudo bash scripts/install.sh` previo si los scripts cambiaron).
- `watson doctor`: `FAIL=0`. WARN aceptables.
- `watson readiness`: `ready_for_simulated_beta1=yes`,
  `ready_for_real_execution=no`, exit 0.
- `watson negative-tests`: `OK=28 FAIL=0`, exit 0.
- `watson contracts check`: exit 0 (registry con 7 candidatas
  válidas, todas `disabled`).
- `watson tools list`: 7 candidatas + `No hay herramientas
  habilitadas.`
- `watson report --out /tmp/openclaw-report.md`: archivo creado
  con `# OpenClaw Report` y `* ready_for_real_execution=no`.
- `release-checklist.sh v0.4.0-alpha`: `FAIL=0`. TODOs
  esperables: ISO y tag (se completan en el PR de build).

## Debe seguir bloqueado

Invariantes que no se relajan en esta release:

- `ready_for_real_execution=no` en `watson readiness`.
- Las 7 tools del registry pueden estar declaradas pero
  `enabled: false`; ninguna `enabled: true`.
- `simulate-execute` no ejecuta herramientas reales: termina en
  `Status: simulated-only`.
- `claw execute` termina en `blocked-by-design` para todo plan
  aprobado.
- `execution manifests` solo documentan la no-ejecución; la
  sección `Result` dice literalmente `No command executed.`
- No hay `sudo` en runtime (`openclaw-stub.sh`,
  `openclaw-policy.sh`, `openclaw-audit.sh`, `workspace.sh`,
  `doctor.sh` sin `repair`).
- No hay red (`curl`, `wget`, `ssh`, sockets) en runtime ni
  instalación.
- No hay runtime real: el motor que ejecutaría herramientas
  todavía no existe.
- No shell libre, no `eval`, no pipes arbitrarios, no
  redirecciones arbitrarias (reglas duras de Beta-1).

## Antes de ISO

PR aparte (no parte de este preflight):

1. **Build ISO** — `scripts/build-iso.sh` (o equivalente
   live-build) que produzca `iso/patrick-os-v0.4.0-alpha.iso` +
   `.sha256`.
2. **Inspección squashfs** — verificar contenido del squashfs:
   Watson presente, scripts presentes con `+x`, configs
   presentes, ningún binario inesperado.
3. **QEMU smoke** — boot live, login, `watson version` reporta
   `v0.4.0-alpha`, `watson doctor` pasa con `FAIL=0`,
   `watson readiness` reporta `ready_for_simulated_beta1=yes`.
4. **GitHub release asset** — subir `.iso` + `.sha256` al
   release de la tag.
5. **Git tag** — `git tag v0.4.0-alpha && git push --tags`
   después del merge del PR de ISO.
