# PatrickOS v0.3.0-alpha checklist

## Estado de la release

- **Preflight `v0.3.0-alpha` listo.** Watson reporta `v0.3.0-alpha`;
  código + docs cerrados. La build de ISO se hace en un PR
  posterior, después de mergear este preflight.
- **OpenClaw Beta-0 cerrado** como capa **dry-run** segura. Ver
  `docs/OPENCLAW_BETA0_CHECKLIST.md`. **El tools registry está
  vacío/deshabilitado** y el **execution gate sigue
  `blocked-by-design`**. **El runtime real NO está implementado.**
- Validación obligatoria antes de tagear: ver "Validación obligatoria".

## Funciones incluidas desde v0.2

- OpenClaw Beta-0 dry-run (`watson claw run "..."`).
- Workspaces locales por modo (`watson ws init/list/path/clean`).
- Plan history + index + viewer + search/recent + filter-tag/priority.
- Plan approval state local (`watson ws approve-plan/reject-plan/plan-status`).
- Policy layer (`configs/openclaw-policy.yaml` + `watson policy check`).
- Tool registry vacío/deshabilitado + viewer (`watson tools list/show/path`).
- Tool contracts documentados (`docs/OPENCLAW_TOOL_CONTRACTS.md`).
- Kill switch local + audit log estructurado + audit summary.
- Execution gate Beta-0 (`watson claw execute`, siempre `blocked-by-design`).
- `watson doctor` (diagnóstico integral) y `watson doctor repair`.
- `install.sh` con verificación post-install (`make check-installed`).
- Detección de repo en doctor para correr desde instalación global.

## Validación obligatoria

```bash
make check
make check-installed
watson doctor
watson policy check
watson tools list
scripts/release-checklist.sh v0.3.0-alpha
```

Lecturas esperadas:

- `make check` → `fails=0`.
- `make check-installed` → `Resultado: OK. Instalación global en sync con el repo.`
- `watson doctor` → `FAIL=0`.
- `watson policy check` → 6 invariantes + `[OK] tool registry: disabled/empty`, `Resultado: OK`.
- `watson tools list` → `"No hay herramientas habilitadas."` (registry vacío/deshabilitado por contrato Beta-0).
- `scripts/release-checklist.sh v0.3.0-alpha` → `FAIL=0`. TODOs esperados (ISO sin construir, tag sin tagear) son normales hasta el build.

Confirmaciones explícitas (no son comandos, son invariantes que el
operador relee antes de avanzar):

- **OpenClaw Beta-0 está cerrado como dry-run.** Ver checklist Beta-0.
- **Tools registry vacío/deshabilitado.** `policy check` lo verifica.
- **Execution gate sigue `blocked-by-design`.** `claw execute` aprobado
  imprime el mensaje y sale en 1; no ejecuta nada.
- **Runtime real NO está implementado.** Ningún path de código corre
  herramientas externas; los gates son la única superficie expuesta.

Complementos opcionales (útiles cuando se sospecha drift):

- `watson audit summary` → muestra el catálogo completo (13 eventos).
- `watson claw run --mode desarrollo --tag doctor --priority high "v0.3 alpha smoke"` → genera plan; el `.md` debe decir `Estado: Dry-run. Nada ejecutado.` y `Permitidas: ninguna`.
- Validar que **NO** existe `iso/patrick-os-v0.3.0-alpha.iso` todavía
  (la build se decide después).

## Antes de construir ISO v0.3 (si se decide)

- Repo limpio, sin PRs abiertos del ciclo.
- README actualizado con cualquier comando nuevo.
- `PROJECT_CONTEXT.md` reflejando el cierre de v0.3.
- Release notes v0.3 finalizadas.
- Decisión documentada en las release notes sobre si la ISO entra o
  no en este alpha.

## No hacer todavía

- OpenClaw runtime con ejecución real de herramientas.
- Red habilitada para OpenClaw.
- `sudo` desde agentes.
- Plugins externos / marketplace.
- NVIDIA propietario.
- Firefox / LibreOffice / OBS preinstalados.
- CI pesado.
- Instalar en hardware real sin QEMU.
