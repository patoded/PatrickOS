# PatrickOS v0.3 — Plan

## Objetivo de v0.3

Pasar de "Watson local útil" a "Watson con preparación agéntica segura":
mantener todo lo que ya funciona (notas, tareas, daily, home, modos,
IA local) y añadir la base sobre la que más adelante podrá enchufarse
OpenClaw como motor agéntico real, sin abrir todavía la puerta a
ejecución libre.

v0.3 es **un release de preparación**, no de runtime agéntico.

## Scope

### En scope

- **Mejorar instalación global.** `install.sh` debe seguir funcionando
  desde el repo y dejar `watson` en PATH sin sorpresas (paths,
  permisos, refresco tras editar `home.sh` u otros scripts). Pensar
  un `make uninstall` simétrico si el costo es bajo.
- **Mejorar home / dashboard.** Iterar sobre `watson home` con feedback
  real de uso diario: secciones, longitudes, ruido. Mantenerlo
  rápido (sin red, sin modelos).
- **Preparar OpenClaw Beta-0.** Ver `OPENCLAW_BETA0_SPEC.md`. En v0.3
  solo entra el contrato y el dry-run; el runtime real **no** se
  prende.
- **Mantener ISO v0.2 estable.** No tocar `iso/` salvo bugfix puntual
  documentado. v0.3 vive en código + docs.

### Fuera de scope (explícito)

- OpenClaw runtime real ejecutando herramientas.
- NVIDIA propietario.
- Firefox / LibreOffice / OBS preinstalados.
- CI pesado.
- Marketplace / plugins externos de cualquier tipo.
- Cambios en el perfil XFCE.

## Estrategia de ISO

Decisión por defecto: **reconstruir ISO v0.3 solo al final**, una vez
que todo el código y docs estén mergeados y validados.

- Cada PR de v0.3 se valida con `make check` y, si toca un script,
  con `watson inicio` / `watson validar` reales.
- La ISO se rearma cuando exista material que valga el costo de
  build (ej. instalación global mejorada + home iterado + Beta-0
  contractualmente listo).
- Si en el medio se cierra un bug crítico de la ISO actual, se
  puede sacar un `v0.2.1-alpha` puntual sin abrir v0.3 de ISO.

## Validación por PR

Cada PR de v0.3 debe pasar:

- `make check` (`fails=0`)
- `watson version` y `watson inicio` sin regresiones visuales
- Bits ejecutables intactos en `scripts/*.sh`
- Sin cambios en `iso/` salvo PR explícitamente etiquetado como ISO

## Cierre de v0.3

Antes de tagear `v0.3.0-alpha`:

- `docs/V0.3_PLAN.md` repasado y cualquier scope movido fuera marcado
  como tal.
- `OPENCLAW_BETA0_SPEC.md` refleja el estado real del contrato (lo
  que está en código vs lo que sigue siendo intención).
- Checklist de release (`scripts/release-checklist.sh v0.3.0-alpha`)
  en verde salvo TODOs esperados.
- Decisión final sobre rebuild de ISO documentada en las release
  notes.
